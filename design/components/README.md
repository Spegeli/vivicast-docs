# Vivicast Components

Status: verbindlich v3

Dieses Verzeichnis beschreibt wiederverwendbare UI-Bausteine.

Komponenten sind keine fachlichen Anforderungen. Sie dienen dazu, Screens konsistent umzusetzen.

## Dateien

- `about-app.md`
- `list-grid-items.md`
- `player.md`
- `settings.md`

## Grundregel

Screens sollen aus gemeinsamen Komponenten aufgebaut werden.

Neue Screen-spezifische Komponenten sind nur sinnvoll, wenn eine vorhandene Komponente nicht passt.

## Abgrenzung

Fachliche Regeln bleiben im PRD.

Screen-Struktur bleibt in `design/screens/`.

Fokus und Navigation bleiben in `design/interaction/`.
